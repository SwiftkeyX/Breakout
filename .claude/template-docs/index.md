# Template Docs Index

These files are structural templates for a new project. They contain no project-specific content.

## How to use
- **Copy and fill** — copy the file to `.claude/docs/`, then replace placeholder content with project-specific details
- **Reference only** — read the template for structural guidance; do not copy (already pre-filled or used as-is)

## Design
| File | Use | Purpose |
|---|---|---|
| `design/systems-design.md` | Copy and fill | System tier table, responsibilities, and dependencies |
| `design/technical-design/_template.md` | Copy per system | Per-system GDD — copy once for each system, rename to `<system>.md` |

## Technical
| File | Use | Purpose |
|---|---|---|
| `technical/architecture.md` | Copy and fill | Script table — one row per script, responsibility, dependencies |
| `technical/technical-preferences.md` | Copy and fill | Engine version, platform, performance budgets, test requirements |
| `technical/best-practices.md` | Copy and fill | Project-critical patterns (top section) + Unity 6 current patterns (pre-filled) |
| `technical/coding-style.md` | Reference only | Required code patterns and anti-patterns |
| `technical/asset-conventions.md` | Reference only | Folder layout, naming rules, import settings |
